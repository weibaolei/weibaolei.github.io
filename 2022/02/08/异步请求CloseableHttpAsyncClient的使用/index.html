# 前言

最近项目有个小需求，需要发送短信验证码，而我们用的短信发送平台是由其他机构提供的，我们只能通过请求前置机并由前置机进行 nginx 转发来调用实际短信平台的 WebService 服务。
由于验证码不会影响业务逻辑，因此我这里开始采用的时异步 http 请求，也就是使用的 httpclient 在 4.0 后提供新的 api CloseableHttpAsyncClient。

# 关于网络调用类型

  1. 传统 BIO
    同步阻塞式 IO，一个连接一个线程，客户端有连接请求时服务器端就需要启动一个线程进行处理
  2. NIO(NOT-Blocking IO)
    同步非阻塞式 IO，一个请求一个线程
  3. AIO
    异步非阻塞式 IO，服务器实现模式为一个有效请求一个线程
